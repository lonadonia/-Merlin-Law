# Merlin.law

A production-ready four-page website for Merlin.law, the medical record review service created by The Guardian Group Civil Services. The site uses a custom editorial design system built around **“The Record, Illuminated”**: deep ink, archival paper, metallic copper, layered record folios, evidence indexes, and restrained document motion.

## Routes

- `/` — Home
- `/how-it-works` — How Merlin Works
- `/capabilities` — Capabilities & Benefits
- `/trust` — Trust, Security, About, FAQ, and Private Demonstration

`/robots.txt`, `/sitemap.xml`, and `/manifest.webmanifest` are generated by the App Router.

## Requirements

- Node.js 20.9 or newer
- npm 10 or newer

## Run locally

```bash
npm install
npm run dev
```

Open `http://localhost:3000`.

The project-local `.npmrc` selects PowerShell for npm scripts so the commands also work from the current Windows directory, whose path contains an ampersand.

## Production build

```bash
npm run lint
npm run build
npm run start
```

The production server opens on `http://localhost:3000` by default.

## Browser QA

The QA script launches the compiled production build, uses the installed Microsoft Edge browser through Playwright, captures desktop/mobile screenshots, checks every route at 1440, 1024, 768, 390, and 360 pixels, and exercises the mobile menu, video modal, FAQ, validation, success state, keyboard skip link, reduced motion, console errors, and horizontal overflow.

```bash
npm run build
npm run qa
```

Screenshots are written to `qa/screenshots/`. To use another Chromium browser:

```powershell
$env:QA_BROWSER_PATH='C:\Program Files\Google\Chrome\Application\chrome.exe'
npm run qa
```

## Explainer video configuration

The single configuration source is `lib/site-config.ts`. Set a direct MP4 or WebM URL with:

```bash
NEXT_PUBLIC_EXPLAINER_VIDEO_URL=https://cdn.example.com/merlin-explainer.mp4
```

Copy `.env.example` to `.env.local` for local configuration. When no URL is provided, the accessible modal intentionally shows a polished storyboard fallback—never a broken player or empty rectangle.

## Demonstration form integration

`components/demo-form.tsx` validates entirely in the browser and intentionally does **not** transmit data. Its preview success message explicitly confirms this. Before launch, connect the successful branch of `submit()` to the client-approved secure form service or server action. Do not add an unapproved endpoint, and do not invite users to upload case documents through the public form.

## Content and factual claims

Primary reusable copy lives in `lib/content.ts`; page-specific editorial copy lives in the related App Router page. Code comments and metadata flag claims requiring final client confirmation. Before publication, verify:

- “Developed by physicians. Built for attorneys.”
- Physician-cost comparison language
- Supported record-set scale and litigation-scale wording
- Closed-system operational descriptions
- The exact legal relationship among Merlin.law, The Guardian Group, and Guardian Group Civil Services
- Any future HIPAA, SOC 2, encryption, or regulatory statements (none are claimed in this build)

## Brand assets

The original archive remains untouched at the project root. Extracted source files are preserved in `source-materials/`, and web-safe copies are in `public/assets/brand/original/`.

The combined Merlin source sheet was not displayed in the site. Exact light and dark crops were created at:

- `public/assets/brand/merlin-lockup-light.png`
- `public/assets/brand/merlin-lockup-dark.png`

The crops preserve the supplied artwork, proportions, color, and baked light/dark backgrounds. `icon-512.png` and `apple-touch-icon.png` are derived from the dark supplied Merlin helmet/shield treatment. Guardian marks are used without redraw, recoloring, or distortion.

## Deployment

Set the canonical production URL first:

```bash
NEXT_PUBLIC_SITE_URL=https://merlin.law
```

For a Node-compatible host, deploy after:

```bash
npm ci
npm run build
npm run start
```

For Vercel, from the project root:

```bash
npx vercel
npx vercel --prod
```

Add `NEXT_PUBLIC_SITE_URL` and, when approved, `NEXT_PUBLIC_EXPLAINER_VIDEO_URL` in the host’s environment settings.
